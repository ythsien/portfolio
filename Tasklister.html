This is the individual assignment of HCI course in Nanyang Technological University in Singapore. In this course, I am requested to follow the Shneiderman's Eight Golden Rules of Interface Design to complete a user interface of a task list application. A set of functionalities are required to achieve in the interface design. The final product is a low fidelity sketch and a high fidelity prototype made by PowerPoint.

Interface Features

This interface contains three views: task view, tag view and trash can. In the task view, current tasks are listed out in a hierarchy way. Users can inspect a certain task by clicking on the task bar and mark the task as complete by clicking the light bulb at the front of each task. Other allowable actions are adding a new task, dragging a task into the trash bin and editing a specific task. When users click on the switch panel on the top, and Gantt chart of tasks will show up. Users can choose whether to show the dependencies of tasks by selecting the checkbox “show dependencies.”

In the tag view, user can manage their self-defined tags they attached to their tasks. By clicking the tag name, the tasks currently having this tag will show up at the right hand side. Users can simply add this tag to other task by the selection menu below.

In the trash can, similar to the tag view, users can manage the deleted tasks. Users can choose to recover a task or delete the task forever. A dialog will pop up for confirmation to prevent users from mis-clicking the “delete forever” button.

Shneiderman's Eight Golden Rules

In this section, I will illustrate how I implement Shneiderman’s eight golden rules of interaction design in my application.

  1. Strive for consistency: the application has a consistent visual layout: a side menu and an upper bar. The color scheme is blue.
  2. Cater to universal usability: it provides feature that allow users to change text language and text size display.
  3. Offer informative feedback: it provides feedback as users complete an action, such as a dialog will pop out when users save a task.
  4. Design dialogs to yield closure: user will see a dialog when an action is completed.
  5. Permit easy reversal of actions: trash can feature allows users to reverse the ‘delete’ action.
  6. Support internal locus of control: it allows users to decide on which page they want to be in.
  7. Reduce short term memory load: actions to enter data and retrieve data are simple so users do not need to remember much steps.
  8. Prevent errors: There is an alert when users do not save a task before clicking the ‘back’ button after they have added or changed the content.

Reflection

In this assignment, there’s more effort put into the detailed design of the user interface. It was nice to know that there are so many details to cater when designing an interface. The eight golden rules help a lot as a reminder for me. However, I feel this assignment is still too specific. The joy of discovering problem and finding solution was taken away since all the requirements were set already.
